פקודות בסיסיות
הפקודות שתשתמשו בהן כל יום. אלה כל מה שצריך כדי לעבוד עם Claude Code.
פקודה מה זה עושה דוגמה
claude הפעלת מצב אינטראקטיבי - שיחה פתוחה claude
claude "task" הרצת משימה חד-פעמית claude "fix the build error"
claude -p "query" שאלה מהירה - מקבלים תשובה ויוצאים claude -p "explain this function"
claude -c חזרה לשיחה האחרונה באותה תיקייה claude -c
claude -r המשך שיחה קודמת (בחירה מרשימה) claude -r
claude commit יצירת Git commit אוטומטית claude commit
/clear ניקוי היסטוריית השיחה /clear
/help הצגת כל הפקודות הזמינות /help
exit / Ctrl+C יציאה מ-Claude Code exit
💡

כל פקודה שמתחילה ב-/ (כמו /clear או /help) היא פקודת מערכת שמורצת בתוך Claude Code עצמו.

⌨
קיצורי מקלדת
קיצורים שחוסכים זמן ועוזרים לנווט מהר יותר.
? הצגת כל קיצורי המקלדת
Tab השלמה אוטומטית של פקודות
↑ היסטוריית פקודות קודמות
/ הצגת כל הפקודות והסקילים
Ctrl+C עצירת פעולה / יציאה
Esc ביטול ההודעה הנוכחית
טיפים למתחילים
✨
טיפים שישדרגו את העבודה
שבעה טיפים שיעזרו לכם לקבל תוצאות טובות יותר מ-Claude Code כבר מהיום הראשון.
1 היו ספציפיים בבקשות
▼
ככל שהבקשה שלכם יותר מדויקת, התוצאה תהיה יותר טובה.
✗ פחות טוב "fix the bug"
✓ הרבה יותר טוב "fix the login bug where users see a blank screen after entering wrong credentials"
2 פרקו משימות מורכבות לשלבים
▼
במקום לתת משימה גדולה אחת, פרקו אותה לצעדים ברורים. למשל, בניית פיצ'ר שלם בצעדים:
1. create a new database table for user profiles 2. create an API endpoint to get and update user profiles 3. build a webpage that allows users to see and edit their information
Claude Code ידע לעבוד על כל שלב בנפרד ולוודא שהכל מתחבר.
3 תנו ל-Claude לחקור קודם
▼
לפני שמבקשים שינויים, תנו ל-Claude להבין את הקוד. קודם - הבנה:
analyze the database schema
ואז - ביצוע:
build a dashboard showing products that are most frequently returned by our UK customers
כשClaude מבין את המבנה קודם, הפתרונות שלו הרבה יותר מדויקים.
4 דברו איתו כמו עם קולגה
▼
אין צורך בפקודות מיוחדות או בשפה טכנית. פשוט תארו מה אתם רוצים בשפה טבעית. דוגמאות לבקשות טבעיות:
what files have I changed?
commit my changes with a descriptive message
add input validation to the user registration form
review my changes and suggest improvements
Claude Code מבין הקשר, כוונה, ושפה טבעית - לא צריך לדבר איתו ב"שפת מכונה".
5 תנו לו יד חופשית - דילוג על אישורים
▼
כברירת מחדל, Claude Code עוצר ומבקש אישור לפני כל פעולה (עריכת קובץ, הרצת פקודה וכו'). אפשר לשחרר אותו לרוץ קדימה - הפעלה במצב "ללא עצירות":
claude --dangerously-skip-permissions
💡

הפלאג הזה מדלג על כל בקשות האישור. מומלץ להשתמש בו רק כשאתם עובדים על פרויקט אישי ולא על production.

אם אתם באמצע שיחה וקיבלתם בקשת אישור, אפשר גם פשוט ללחוץ על a (always allow) כדי לאשר את כל הפעולות מאותו סוג לכל הסשן.
6 הטרמינל נפתח ישר על Claude Code
▼
נמאס לכם להקליד claude בכל פעם שאתם פותחים טרמינל? אפשר להגדיר שהוא ייפתח אוטומטית.

Mac - הוסיפו שורה אחת לסוף הקובץ ~/.zshrc:
echo 'claude' >> ~/.zshrc
או - פרופיל ייעודי ב-Terminal של Mac:
1. פתחו Terminal → Settings → Profiles
2. צרו פרופיל חדש בשם "Claude Code"
3. בלשונית Shell, בשדה "Run command" הכניסו: claude
4. הגדירו את הפרופיל הזה כברירת מחדל
Windows - הוסיפו פרופיל ב-Windows Terminal:
1. פתחו Windows Terminal → Settings → Add new profile
2. בשדה Command line הכניסו: claude
3. הגדירו כ-Default profile
7 תתייעצו עם Claude לפני שבונים סקיל
▼
סקילים (Skills) הם "מתכונים" שאתם מלמדים את Claude Code - הוראות ארוזות שהוא מפעיל עם פקודה אחת. לפני שבונים סקיל חדש, פשוט שאלו אותו ישירות:
how do I create a custom skill?
what are skills and how do I use them?
can you help me build a skill that does X?
ל-Claude Code יש מדריך מובנה (claude-code-guide) שיודע להסביר בדיוק איך בונים סקילים, מה המבנה הנכון, ואיך מפעילים אותם. אתם פשוט שואלים - והוא מלמד אתכם.
⚡

הכלל: כל פעם שאתם לא יודעים איך לעשות משהו ב-Claude Code - פשוט תשאלו אותו. הוא יודע להסביר על עצמו.
