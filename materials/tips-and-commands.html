פקודות בסיסיות
הפקודות שתשתמשו בהן כל יום. אלה כל מה שצריך כדי לעבוד עם Claude Code.
פקודה מה זה עושה דוגמה
claude הפעלת מצב אינטראקטיבי - שיחה פתוחה claude
claude "task" הרצת משימה חד-פעמית claude "fix the build error"
claude -p "query" שאלה מהירה - מקבלים תשובה ויוצאים claude -p "explain this function"
claude -c חזרה לשיחה האחרונה באותה תיקייה claude -c
claude -r המשך שיחה קודמת (בחירה מרשימה) claude -r
claude commit יצירת Git commit אוטומטית claude commit
/clear ניקוי היסטוריית השיחה /clear
/help הצגת כל הפקודות הזמינות /help
exit / Ctrl+C יציאה מ-Claude Code exit
💡

כל פקודה שמתחילה ב-/ (כמו /clear או /help) היא פקודת מערכת שמורצת בתוך Claude Code עצמו.

⌨
קיצורי מקלדת
קיצורים שחוסכים זמן ועוזרים לנווט מהר יותר.
? הצגת כל קיצורי המקלדת
Tab השלמה אוטומטית של פקודות
↑ היסטוריית פקודות קודמות
/ הצגת כל הפקודות והסקילים
Ctrl+C עצירת פעולה / יציאה
Esc ביטול ההודעה הנוכחית
💡
מה אני יכול לעשות עם זה?
5 תרחישים קונקרטיים שאנשי Product Ops יכולים לנסות מחר בבוקר.
📊
ניתוח דאטה מהיר
יש לכם CSV עם אלפי שורות של נתוני שימוש? תנו ל-Claude Code לנתח, לסנן ולמצוא תובנות - בלי לפתוח Python, בלי לכתוב קוד.
🔄
אוטומציה של דוחות
ליצור סקריפט שמושך נתונים ממקור (Jira, Google Sheets, API) ומייצר דוח שבועי אוטומטי - במקום לעשות את זה ידנית כל שבוע.
🛠️
בניית כלים פנימיים
לבנות דשבורד פנימי פשוט, מחשבון ROI, או כלי השוואה - בלי לחכות לתור בפיתוח.
🧹
ניקוי ועיבוד דאטה
לנקות, לאחד ולתקנן נתונים ממקורות שונים - הוצאות מ-Excel, אירועים מ-Mixpanel, tickets מ-Jira - לקובץ אחד מסודר.
⚡
Prototyping מהיר
לבנות POC של פיצ'ר כדי להדגים אותו ל-stakeholders - ממסמך דרישות לדמו עובד, תוך שעות ולא שבועות.
טיפים למתחילים
✨
טיפים שישדרגו את העבודה
שבעה טיפים שיעזרו לכם לקבל תוצאות טובות יותר מ-Claude Code כבר מהיום הראשון.
1 דברו איתו כמו עם קולגה
▼
אין צורך בשפה טכנית. פשוט תארו מה אתם צריכים - בדיוק כמו שהייתם מסבירים לקולגה חכם. דוגמאות מהעולם של ProdOps:
what's in this CSV file?
summarize the main trends in this data
review this PRD and flag potential issues
what should I prioritize first in this backlog?
Claude Code מבין הקשר, כוונה ושפה טבעית - לא צריך לדבר איתו ב"שפת מכונה".
2 תתחילו עם שאלה פשוטה
▼
הדרך הכי טובה להתחיל היא פשוט לפתוח תיקייה עם קובץ - ולשאול שאלה ישירה:
what files are here and what's in them?
summarize what's in this CSV
what does this project do?
⚡

מה שעובד - לקחת משהו קטן מהעבודה ולתת ל-Claude Code לגעת בו. מסמך, טבלה, בעיה שחוזרת - לא משנה מה.

3 היו ספציפיים בבקשות
▼
ככל שהבקשה שלכם יותר מדויקת, התוצאה תהיה יותר טובה.
✗ פחות טוב "analyze this data"
✓ הרבה יותר טוב "look at user_events.csv, find which step in the onboarding flow has the highest drop-off rate, and show me the top 3 issues"
✗ פחות טוב "create a report"
✓ הרבה יותר טוב "create a weekly sprint summary from the Jira export - group by assignee, show completed vs open, highlight anything overdue"
4 תנו ל-Claude לחקור קודם
▼
לפני שמבקשים לבנות משהו, תנו ל-Claude להבין את הנתונים קודם. קודם - הבנה:
what columns are in this dataset and what does each one mean?
ואז - ביצוע:
build a dashboard showing feature adoption rates by cohort, using the data in usage_events.csv
כש-Claude מבין את המבנה קודם, הפתרונות שלו הרבה יותר מדויקים.
5 פרקו משימות מורכבות לשלבים
▼
במקום לתת משימה גדולה אחת, פרקו אותה לצעדים ברורים:
1. load the sales_q1.csv file and check its structure 2. calculate revenue by product category and month 3. create a bar chart showing month-over-month trends
Claude Code ידע לעבוד על כל שלב בנפרד ולוודא שהכל מתחבר.
6 תתייעצו עם Claude לפני שבונים סקיל
▼
סקילים (Skills) הם "מתכונים" שאתם מלמדים את Claude Code - הוראות ארוזות שהוא מפעיל עם פקודה אחת. לפני שבונים סקיל חדש, פשוט שאלו אותו ישירות:
how do I create a custom skill?
can you help me build a skill that does X?
⚡

הכלל: כל פעם שאתם לא יודעים איך לעשות משהו ב-Claude Code - פשוט תשאלו אותו. הוא יודע להסביר על עצמו.

7 דילוג על אישורים (מתקדם)
▼
כברירת מחדל, Claude Code עוצר ומבקש אישור לפני כל פעולה. אחרי שמכירים את הכלי ורוצים שיריץ משימות בלי הפסקות - אפשר לאשר כל סוג פעולה בנפרד באמצע שיחה על ידי לחיצה על a (always allow).

לאלה שרוצים לדלג על הכל מההתחלה:
claude --dangerously-skip-permissions
💡

מומלץ רק לפרויקטים אישיים. לא להשתמש על קבצי production או נתוני חברה רגישים.

האתגר שלכם להיום

פתחו טרמינל, הכנסו לתיקייה עם קובץ כלשהו, הקלידו claude ושאלו: "what's in this folder?" - ותראו מה קורה.
